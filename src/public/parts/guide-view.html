Guide

How to use BOIAuto

1

Add a Bot

Go to the Bot page from the sidebar and click + Add. Enter an optional name (defaults to bot id), pick a Type, then click Generate Token. A secret token will appear below with copy + connect instructions.

Dual Allows a maximum of 2 access tokens per bot.
Shared Unlimited access tokens.

After creating the bot, the button changes to Waiting Later with a spinner. When your bot client connects, it becomes Finished.

2

Connect a Bot Client

Copy the bot token from the Add Bot modal (or edit it later in the Bot page). Set BOT_TOKEN in your bot client env, or store it in bot.config.json. Run bun run bot:start. The bot's status will switch from Connecting to Connected once the client is online.

3

Generate Access Tokens

On the Access page, click + Generate to create an access token tied to a specific bot. Pick a Type:

Private Single-use access. Max 1 automate per token. Shown as 0/1 or 1/1.
Shared Unlimited automates. Usage shown as a single number (e.g. 3).
Business Single-use access + price/day (game currency). Max 1 automate.

Share the access token with the recipient. They can use it to run automates linked to that bot.

4

Add an Automate

On the Automate page, click + Add. Enter your Bearer token, then select an access token from the dropdown (not a bot directly). Each access item shows its usage (0/1 for Private/Business, single number for Shared) and type badge. The access type is inherited by the automate.

5

Toggle Automation Features

Each automate card has 3 toggles:

Skill — auto-upgrade skill levels
Training — auto-train units
Work — auto-perform work tasks

When any feature is ON, the card border glows green or blue.

6

Dashboard Overview

The Dashboard page (first item in the sidebar) shows real-time stats: total automates, total bots, total access tokens, connected bots, automation status (Skill/Training/Work counts), and total balance + diamond across all automates. Below the stats are quick-action cards that link to each section, and a list of the latest 5 bots with their connection status.

Tip

All tokens are stored in SQLite. In dev mode the DB is in-memory and reset on restart; in production it persists to data.db.
